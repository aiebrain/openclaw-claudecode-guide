2. 프론트엔드 — Lovable + Vercel

2-1. Lovable이란?

문장(프롬프트)을 입력하면 AI가 React + TailwindCSS 코드를 자동 생성해주는 서비스입니다.

코딩을 전혀 몰라도 프로페셔널한 쇼핑몰 화면을 만들 수 있습니다.

Lovable의 강점

- 프롬프트 → 즉시 화면 생성 (30초 이내)
- 수정도 프롬프트로 ("헤더 색상 검정으로 바꿔줘")
- 반응형(모바일) 자동 지원
- GitHub 자동 연동 → Vercel 배포까지 원클릭

2-2. Lovable 시작하기

  1. lovable.dev 가입
  2. "New Project" 클릭
  3. 아래 프롬프트를 입력:
한국어 쇼핑몰을 만들어줘.

필요한 페이지:
- 메인 (히어로 배너 + 인기 상품 그리드 4열)
- 상품 목록 (카테고리 필터 + 검색바)
- 상품 상세 (이미지 갤러리 + 옵션 선택 + 장바구니 담기)
- 장바구니 (수량 변경 + 삭제 + 총액 계산)
- 결제 페이지 (배송지 입력 + 결제수단 선택)
- 마이페이지 (주문 내역 + 배송 추적)
- 로그인/회원가입 (카카오/구글 소셜 로그인)

디자인:
- 미니멀, 화이트 배경
- 라운드 카드 (border-radius: 12px)
- 모바일 우선 반응형
수정 프롬프트 예시

``
- 헤더에 로고를 왼쪽으로, 장바구니 아이콘을 오른쪽에 추가해줘
- 상품 카드에 할인율 빨간 뱃지 추가해줘 (예: -30%)
- 장바구니 아이콘에 담긴 상품 수 뱃지 표시해줘
- 푸터에 고객센터 전화번호와 이메일 추가해줘
``

2-3. Vercel 배포

단계:

  1. Lovable → "Deploy" 버튼 → GitHub 자동 push
  2. vercel.com 가입 (GitHub 계정으로)
  3. "Import Project" → Lovable이 만든 레포 선택
  4. "Deploy" 클릭 → 30초 내 배포 완료!
Vercel 무료 플랜

- 월 100GB 트래픽 (소규모 쇼핑몰 충분)
- HTTPS(SSL) 자동 적용
- 글로벌 CDN (전 세계 빠른 로딩)
- Preview 배포 (수정 사항 라이브 전 미리 확인)

2-4. 도메인 연결

  1. 도메인 구매: 가비아, 후이즈 (연 1~2만원)
  2. Vercel → Settings → Domains → 구매한 도메인 입력
  3. 도메인 DNS 설정에서 CNAME 레코드 추가:
  4. 호스트: @ 또는 www
  5. 값: cname.vercel-dns.com
  6. 5~30분 후 SSL 자동 적용 → https://yourbrand.com 접속 가능!

3. 백엔드 — Supabase

3-1. Supabase란?

Firebase의 오픈소스 대안으로, 데이터베이스 + 인증 + 파일 저장을 한 곳에서 제공합니다.

왜 Supabase?

- 무료 플랜: DB 500MB + Storage 1GB (시작 충분)
- 카카오/구글 소셜 로그인 내장
- 실시간 데이터 동기화 (주문 상태 즉시 반영)
- SQL Editor 내장 (복붙만으로 테이블 생성)

3-2. 프로젝트 생성

  1. supabase.com 가입 (GitHub 계정)
  2. "New Project" → 이름 입력
  3. 리전: Northeast Asia (Tokyo) 선택 (한국에서 가장 빠름)
  4. DB 비밀번호 설정 → "Create"

3-3. 데이터베이스 스키마

복붙만 하면 됩니다

아래 코드를 이해할 필요 없어요. Supabase SQL Editor에 붙여넣고 실행하면 테이블이 자동으로 만들어집니다.

회원 테이블:

CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  name TEXT,
  phone TEXT,
  address TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

상품 테이블:

CREATE TABLE products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT,
  price INTEGER NOT NULL,
  sale_price INTEGER,
  category TEXT,
  image_urls TEXT[],
  stock INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

주문 테이블:

CREATE TABLE orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  status TEXT DEFAULT 'pending',
  total_amount INTEGER NOT NULL,
  shipping_address TEXT,
  payment_method TEXT,
  payment_key TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

3-4. 소셜 로그인 설정

카카오 로그인

  1. developers.kakao.com → 앱 생성
  2. 카카오 로그인 → 활성화
  3. Redirect URI 등록: https://YOUR_PROJECT.supabase.co/auth/v1/callback
  4. Supabase → Auth → Providers → Kakao → ID/Secret 입력
소셜 로그인 왜 중요한가?

일반 회원가입 전환율: 약 20% / 카카오 로그인: 약 60~70% (3배 이상!)

3-5. Storage (파일 저장)

  1. Supabase → Storage → New Bucket → product-images
  2. Public 체크 (상품 이미지 공개 접근)
  3. 파일 크기 제한: 5MB

3-6. Edge Functions

함수 역할
handle-payment Toss/래피드 결제 확인 후 주문 상태 업데이트
check-stock 결제 전 재고 확인 + 결제 후 재고 차감
send-notification 주문 확인 후 알리고 API로 알림톡 발송
auto-reply-review Claude API로 리뷰 자동 답글

4. 결제 연동 — Toss + 래피드

4-1. 결제 분기 구조

고객이 "결제하기" 클릭
  │
  ├─ 🇰🇷 국내 일반 결제 → Toss Payments
  │   ├─ 신용/체크카드
  │   ├─ 카카오페이
  │   ├─ 네이버페이
  │   └─ 계좌이체
  │
  └─ 📚 디지털 콘텐츠 판매 → 래피드(Latpeed)
      ├─ 전자책/PDF
      ├─ 온라인 강의
      ├─ 디지털 파일
      └─ 멤버십/구독

4-2. Toss Payments (국내 일반 결제)

국내 결제 1위

카드/카카오페이/네이버페이/계좌이체 — 한국 고객 결제의 95% 커버

연동 단계:

  1. developers.tosspayments.com 가입
  2. 사업자등록번호 인증 → API 키 발급
  3. 클라이언트 키 → 프론트엔드 / 시크릿 키 → Edge Function

수수료:

결제 수단 수수료
신용카드 3.3%
카카오페이 3.3%
네이버페이 3.3%
계좌이체 1.5%

4-3. 래피드(Latpeed) — 디지털 콘텐츠 판매

사업자 등록 없이 3분 만에 판매 시작

전자책, 디지털 파일, 온라인 강의, 멤버십 판매에 특화된 1인 기업용 판매 도구

래피드의 강점:

  • 사업자 등록/심사 없이 바로 판매 가능
  • 링크 하나로 결제 + 자동 발송
  • 멤버십/구독 결제 지원
  • 구글폼 설문 + 결제 동시 처리
  • 웹훅/구글시트/노션 연동

시작하기:

  1. latpeed.com 가입 (간단 인증)
  2. 상품 등록 (제목 + 가격 + 파일 업로드)
  3. 판매 링크 생성 → 공유 → 끝!
래피드 vs Toss 사용 시점

- 실물 상품 (의류, 식품 등) → Toss Payments
- 디지털 상품 (전자책, 강의, 파일) → 래피드
- 혼합 판매 → 둘 다 연동

6. AI 운영 자동화

핵심

Claude Code로 자동화 코드를 직접 구현하고, Claude API로 AI 기능을 연동합니다.

6-1. 전자책 자동 발송 (센드맨)

🛒 고객이 전자책 결제 완료
  → Cafe24 센드맨이 자동 감지
  → 파일/링크/코드 즉시 발송 (이메일/문자)
  → 배송완료 자동 처리
  → CS 문의 대폭 감소!
센드맨 활용 예시

- 전자책 PDF 자동 발송
- 수강 링크 자동 전달
- 라이선스 코드 자동 발급
- 디지털 다운로드 링크 발송

6-2. 리뷰 답글 자동화 (Claude Code)

Claude Code로 직접 구현 가능

Supabase의 리뷰 테이블을 모니터링하고, 새 리뷰가 등록되면 Claude API로 답글을 자동 생성하는 Edge Function을 만듭니다.

⭐ 고객이 리뷰 작성 (Supabase INSERT 트리거)
  → Edge Function 실행
  → Claude API에 리뷰 전달
  → "이 리뷰에 친절하고 전문적인 답글을 작성해줘"
  → 답글 자동 저장 → 고객에게 노출
Claude 답글 예시

고객 리뷰: "배송 빠르고 품질 좋아요! 다만 색상이 사진이랑 살짝 달라요"
AI 답글: "소중한 리뷰 감사합니다! 😊 빠른 배송과 품질에 만족해주셔서 기쁩니다. 색상 차이에 대해 불편을 드려 죄송해요. 모니터 설정에 따라 약간의 차이가 있을 수 있는데, 더 정확한 색상 표현을 위해 촬영 환경을 개선하고 있습니다."

6-3. 상품 설명 자동 생성 (Claude Code)

📦 새 상품 등록 (이름 + 카테고리만 입력)
  → Claude API 호출
  → "이 상품의 매력적인 설명을 작성해줘. SEO 키워드 포함"
  → products.description 자동 업데이트

6-4. 재고 알림 자동화

📊 재고 5개 이하 감지 (Supabase 트리거)
  → Edge Function → 알리고 알림톡
  → "⚠️ [상품명] 재고 3개 남음 — 발주 필요"

7. 구축 로드맵 (2주)

하루 2~3시간씩 진행 기준

일차 작업 도구 예상 시간

9. 자주 묻는 질문 (FAQ)

Q. 코딩을 전혀 모르는데 정말 가능한가요?

네. Lovable이 UI를, Supabase가 백엔드를 자동 처리합니다. SQL도 복사-붙여넣기만 하면 됩니다.

[!question] Q. 사업자등록증이 꼭 필요한가요?
Toss Payments 연동 시 필수입니다. 단, 래피드는 사업자 없이 판매 가능!

[!question] Q. 전자책/디지털 파일은 어떻게 판매하나요?
래피드로 결제 링크를 만들거나, 센드맨으로 Cafe24에서 결제 후 자동 발송합니다.

[!question] Q. Cafe24나 Shopify 대비 장점은?
월정액 없이 시작 가능하고, 디자인/기능 커스터마이징이 완전히 자유롭습니다.

[!question] Q. 리뷰 답글 자동화는 어떻게 하나요?
Claude Code로 Supabase Edge Function을 만들면 됩니다. 새 리뷰 → Claude API → 자동 답글 저장.
